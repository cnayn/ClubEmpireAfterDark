# Nightclub Feature Design Skill

Use this skill before implementing any new gameplay feature.

## Feature design checklist

For every feature, define:

1. Player fantasy
What does the player feel they are doing?

2. Gameplay purpose
What problem, choice, or progression goal does it create?

3. Core loop connection
How does it affect day preparation or night results?

4. Inputs
What does the player choose?

5. Outputs
What changes after the night simulation?

6. Economy impact
How does it affect cash, costs, revenue, reputation, guests, loyalty, VIPs, or risk?

7. Failure state
How can it go wrong?

8. MVP version
What is the smallest shippable version?

9. Tests
What behavior needs unit tests?

10. Docs
Which docs must be updated?

## Game-specific rules

The game is an offline-first 2D nightclub management sim.
Do not design backend, multiplayer, cloud save, ads, or IAP for MVP.
Do not use real celebrity names.
Risky nightlife systems must be fictional, satirical, or represented as compliance/reputation/business risk.

## Preferred feature order

1. Named staff.
2. Staff traits.
3. Events / themed nights.
4. Regular customers.
5. VIP system.
6. City/district progression.
7. Visual polish and animation.
8. Audio/music flavor.
